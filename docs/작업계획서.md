## Phase 2.5 - MVP 검증 (소규모 테스트) 🧪
- [ ] **크루 생성 → 미션 인증 → 수동 리워드** 플로우만으로 10~20명 테스트
- [ ] **포인트/결제 없이** 크루 구조 자체에 대한 사용자 반응 검증
- [ ] 피드백 수집 후 Phase 3 방향 조정

## Phase 3 - Supabase & 인증 백엔드 ⬜
- [ ] **DB 스키마:**
  - `users`, `crews`, `crew_members` (코어)
  - `missions`, `mission_verifications` (미션 인증)
  - `user_points`, `point_logs` (포인트 관리)
  - `reward_redemptions` (리워드 교환)
  - `crew_revenue_share` (수익 배분 계약)
  - `expense_requests` (지출 승인 관리)
  - `settlement_logs` (정산서 업로드 및 검증)
- [ ] **Auth:** 소셜 로그인(카카오, 구글) + 본인인증(KYC) 단계적 도입
- [ ] **크루/미션 CRUD:** API Routes 구현

## Phase 4 - 결제 & 포인트 시스템 ⬜
- [ ] **토스페이먼츠 간편결제 연동:** 참여금 결제 (소액 최적화)
- [ ] **포인트 적립 로직:** 미션 달성도 기반 단계별 적립
- [ ] **마이 월렛 UI:** 보유 포인트, 참여금 현황, 리워드 내역
- [ ] **리스크 락(Lock):** 결제 후 7일 경과 + 미션 80% 달성 시에만 리워드 활성화
- [ ] **포인트 상점:** 기프티콘 목록 및 교환 신청

## Phase 5 - 크루 운영 고도화 & 투명성 시스템 ⬜
- [ ] **크루 전용 실시간 채팅:** Supabase Realtime 기반 크루 내 소통 + 파일 업로드(정산서/영수증)
- [ ] **수익 배분 협상 도구:** 크루장 배분율 제안 → 전원 동의 시 확정, `crew_revenue_share` 테이블 관리
- [ ] **지출 승인 시스템:** 크루장 지출 항목 등록 → 참여자 승인/반려 + 영수증 증빙 필수
- [ ] **투명성 대시보드:** 누적 매출·지출·인당 예상 배분액 실시간 차트
- [ ] **미션 템플릿 시스템:** 카테고리별 프리셋 미션 제공 (수동 → 추후 AI 전환)
- [ ] **크루장 매니징 툴:** 미션 승인/반려, 크루원 통계 대시보드
- [ ] **상호 인증 시스템:** 크루원 간 인증 피드백 시 보너스 포인트
- [ ] **이의제기 시스템:** 크루장-크루원 간 분쟁 조정 (48시간 소명 → 강제 정산)
